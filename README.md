# Miterion

Clinical Operations Intelligence platform. Quickly structure public registry data into useful operational snapshots.

## Known MVP Gaps

1. **Snapshot Lead Save**: The lead save route for snapshot generation is not implemented yet. Before implementing PDF generation, add an API route to save leads to `leads_snapshot`.

2. **Rate Limiting**: Rate limiting is not implemented yet. Add either:
   - IP-based rate limiting in API routes
   - Upstash Redis-based rate limiting

These features should be implemented before production deployment.

## Local Setup

### Prerequisites

- Node.js 18+
- npm
- Supabase project

### Installation

1. Install dependencies
```bash
npm install
```

2. Copy `.env.example` to `.env.local` and configure:
```bash
cp .env.example .env.local
```

Required environment variables:
- `NEXT_PUBLIC_SUPABASE_URL`: Your Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Public anon key (limited access)
- `SUPABASE_SERVICE_ROLE_KEY`: Admin access key (keep secret!)
- `RESEND_API_KEY`: Resend.com API key for emails
- `RESEND_FROM_EMAIL`: Sender email address
- `NEXT_PUBLIC_SITE_URL`: Site URL (http://localhost:3000 for development)

3. Start the development server
```bash
npm run dev
```

### Supabase Setup

1. Create tables using this SQL:

```sql
-- Buckets: create 'rfp-uploads' (private) and 'snapshots' (private)
-- Allowed MIME for rfp-uploads: application/pdf, application/vnd.openxmlformats-officedocument.spreadsheetml.sheet

create table leads_snapshot (
 id bigint generated by default as identity primary key,
 email text not null,
 company text,
 indication text not null,
 phase text not null,
 geo text not null,
 pdf_path text,
 created_at timestamptz default now()
);
create index on leads_snapshot(email);

create table leads_rfp (
 id bigint generated by default as identity primary key,
 email text not null,
 company text,
 message text,
 created_at timestamptz default now()
);

create table uploads (
 id bigint generated by default as identity primary key,
 lead_id bigint references leads_rfp(id) on delete cascade,
 file_path text not null,
 file_size int,
 mime_type text,
 created_at timestamptz default now()
);

create table leads_dct (
 id bigint generated by default as identity primary key,
 email text not null unique,
 created_at timestamptz default now()
);

create table api_cache (
 id bigint generated by default as identity primary key,
 cache_key text not null unique,
 payload jsonb not null,
 created_at timestamptz default now()
);
create index on api_cache(cache_key);

create table email_queue (
 id bigint generated by default as identity primary key,
 to_email text not null,
 subject text,
 html text,
 sent boolean default false,
 created_at timestamptz default now()
);

-- RLS: enabled on all tables, no policy = default DENY for anon and auth.
-- Backend uses SUPABASE_SERVICE_ROLE_KEY, which bypasses RLS.
alter table leads_snapshot enable row level security;
alter table leads_rfp enable row level security;
alter table uploads enable row level security;
alter table leads_dct enable row level security;
alter table api_cache enable row level security;
alter table email_queue enable row level security;
```

2. Create Storage buckets:
- `rfp-uploads`: Private bucket for RFP file uploads
- `snapshots`: Private bucket for generated PDF reports

3. Configure MIME types for `rfp-uploads`:
- `application/pdf`
- `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`

### Row Level Security (RLS)

All tables have RLS enabled with no policies, defaulting to DENY for both anonymous and authenticated users. The backend uses `SUPABASE_SERVICE_ROLE_KEY` which bypasses RLS entirely.

This is intentional as all data access should go through our validated API routes.

### Deployment

#### Vercel

1. Create a new project and connect to your repository

2. Configure environment variables:
   - Add all variables from `.env.example`
   - Set `NEXT_PUBLIC_SITE_URL` to your production domain

3. Deploy settings:
   - Framework Preset: Next.js
   - Node.js Version: 18.x
   - Build Command: `next build`
   - Output Directory: `.next`

4. Deploy! The project will auto-detect all settings including next-intl localization.

#### Important Notes

- The backend uses ClinicalTrials.gov API v2 which is public and requires no API key
- PDF generation uses `@sparticuz/chromium-min` optimized for Vercel
- Files are stored in Supabase Storage (not local filesystem)
- Emails are sent via Resend with queue fallback in database
- The app supports English (`/`) and Polish (`/pl`) routes
- All user-facing strings are in `messages/{locale}.json`
