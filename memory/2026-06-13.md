## Issue: Waitlist Form Input Visibility Problem - FIXED
- **Problem:** The cursor and text were not visible in the email input field on the waitlist form. The text and cursor appeared to be white on a white background.
- **Root Cause:** The input field was inheriting `text-white` from its parent `.bg-navy` container, making white text on white background invisible.
- **Fix Applied:**
  1. Added explicit `text-navy` to `.input-field` class
  2. Added specific rule for input fields within dark backgrounds: `.bg-navy .input-field, .text-white .input-field { @apply text-navy bg-white; }`
  3. Added `caret-color: var(--navy);` to ensure cursor visibility
- **Files Modified:** `src/app/globals.css`
- **Status:** ✅ RESOLVED
