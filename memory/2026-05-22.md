# 2026-05-22 - Email Sending Bug: Resend 403 Forbidden

**Issue:** Email notifications were failing in the local environment with a `403 Forbidden` error from Resend.

**Confirmed Facts:**
- **API Key:** Valid.
- **Failing Environment:** Local/workspace runtime.
- **Failing Routes:** `src/app/api/rfp-submit/route.ts` and `src/app/api/dct-waitlist/route.ts`.
- **Error Message:** `"The send.miterion.com domain is not verified."`

**Root Cause:**
The application was attempting to send emails from the domain `send.miterion.com`, which is not verified in the Resend account associated with the current `RESEND_API_KEY`. This is a configuration issue in the Resend dashboard, not a code or API key error.

**Resolution Steps:**
1.  Verify `send.miterion.com` in the Resend dashboard by adding the required DNS records.
2.  Alternatively, update the `RESEND_FROM_EMAIL` to use a domain that is already verified.
