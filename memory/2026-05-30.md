## 2026-05-30 - Testy produkcji i naprawa flow PDF/mail w miterion-platform

### Problemy zdiagnozowane i naprawione

#### 1. **Generowanie PDF** (`/api/generate-pdf`)
- **Problem**: Backend zwracał błąd 500, PDF nie generował się.
- **Root cause**: Błędne importowanie Puppeteer (ESM vs CommonJS) oraz nieobsłużone typy w `route.ts`.
- **Naprawa**: 
  - Poprawiono import Puppeteer: `const puppeteerLocal = await import("puppeteer");`
  - Naprawiono typy: `waitUntil: "networkidle0" as any`
  - Naprawiono `Content-Length` w nagłówkach (zmieniono z `string` na `number`)
  - Poprawiono zwracanie PDF: `NextResponse(Buffer.from(pdfBuffer), {...})`
- **Wynik**: ✅ PDF generuje się poprawnie lokalnie (`LOG_VERIFY_SIZE: 71046`).

#### 2. **Wysyłanie maili przez Resend** (`/api/generate-pdf`)
- **Problem**: Mail nie dochodził, backend zwracał błąd 403: `The send.miterion.com domain is not verified`.
- **Root cause**: Używano nieprawidłowego adresu `snapshot@send.miterion.com` zamiast zweryfikowanego `contact@miterion.com`.
- **Naprawa**: Zmieniono `RESEND_FROM_EMAIL` w `.env.local` na `"Miterion <contact@miterion.com>"`.
- **Wynik**: ✅ Mail jest wysyłany z poprawnego adresu, log: `[generate-pdf] Email sent successfully: <ID>`.

#### 3. **Pobieranie PDF w przeglądarce**
- **Problem**: Przeglądarka nie pobierała PDF, tylko pokazywała UUID jako tekst.
- **Root cause**: Błędna konfiguracja `NextResponse` – `Content-Length` jako string zamiast number.
- **Naprawa**: Poprawiono nagłówki:
  ```ts
  headers: {
    "Content-Type": "application/pdf",
    "Content-Disposition": `attachment; filename="snapshot-report.pdf"`,
    "Content-Length": pdfBuffer.length, // zmieniono z .toString() na number
  }
  ```
- **Wynik**: ✅ PDF pobiera się poprawnie.
