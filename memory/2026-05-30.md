
## 2026-05-30 - Email Notification Fix

**Issue:** The user reported that notifications to `contact@miterion.com` were not being sent.

**Root Cause:** The `RESEND_FROM_EMAIL` was incorrectly set to `snapshot@send.miterion.com`, which is not a verified domain in Resend. The correct verified sender is `contact@miterion.com`.

**Fix:**
1. Reverted `RESEND_FROM_EMAIL` in `.env.local` to `"Miterion <contact@miterion.com>"`.
2. Added `bcc: "contact@miterion.com"` to the email in `src/app/api/generate-pdf/route.ts` to ensure a copy is sent to the admin.
3. Added `bcc: 'contact@miterion.com'` to the email in `src/app/api/dct-waitlist/route.ts` to ensure a copy is sent to the admin.

**Verification:** The changes were made, but the server needs to be restarted to apply the new environment variables. The user should test the endpoint again to confirm the email is sent.
